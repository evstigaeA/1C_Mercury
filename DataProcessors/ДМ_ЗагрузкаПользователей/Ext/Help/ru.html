Обработка выполняет создание новых пользователей информационной базы и групп доступа для них.

Загрузка данных выполняется из файла Excel. Пример файла ниже:

Cl. Plnt Name 1 ValA CustNoPlnt Vendor number plant Cal Name 2 Street and House Number PO Box PostalCode City
100 5100 5100 RU DR HQ 5100 0050011670 Z0 ЦО-КЮМ Москва RigaLand ул. Вятская, корп. 13-14 д. 27 127015 Москва
100 5101 5100 RU PL Petmol 5101 0050004989 0010005519 Z0 ПР Петмол 6-ой Верхний переулок 1,литера 194292 Санкт-Петербург

Перед выполнением загрузки данных необходимо установить настройки выполнения обработки:

  • Начальный номер строки - номер первой строки с данными для загрузки, по умолчанию - 2;
  • Сокращать наименования - нужно ли исключать из наименований создаваемых элементов справочников первые 7 символов (например: "5100 RU " может быть удалено при загрузке);
  • Создавать организации - если по номеру планта организация не будет найдена, то создавать новый элемент;
  • Создавать предприятия - если по номеру планта предприятие не будет найдено, то создавать новый элемент.

 

Номер планта получается из 2-й колонки (Plnt).

Имя создаваемого пользователя и логин для входа формируется из значений колонок Plnt и Name1.

Поиск организаций в справочнике Организации выполняется по реквизиту "Код SAP" по значению колонки Plnt.

Поиск предприятий в справочнике Предприятия выполняется по реквизиту "Plant" по значению колонки Plnt.

При создании нового пользователя ему сразу же генерируется случайный пароль и устанавливаются признаки "Аутентификация 1С:Предприятия" и "Показывать в списке выбора".

По имени пользователя ищется одноименная группа доступа. Если такой группы нет, то создается новая группа доступа с профилем доступа "Пользователь" и ограничениями: "Организации Все запрещены, кроме 1 значения". В качестве разрешенной организации устанавливается найденная по номеру планта организация.

Для списка загруженных пользователей можно вызвать форму обработки для повторной пакетной генерации паролей.

Обработка служит для создания скрипта PowerShell для проверки наличия и актуальности содержимого файла-индикатора состояния системы автообменов. Запуск скрипта устанавливается администратором сервера Windows в библиотеке планировщика заданий.
Если регламентные задания обмена данными с Меркурием не выполняются, то файл-индикатор успешного выполнения заданий отсутствует либо его содержимое устарело (дата+время последнего успешного выполнения регламентных заданий меньше текущего времени более, чем на 15 минут).

Параметры генерации скрипта:

Путь к файлу-индикатору Полный путь к файлу-индикатору, обновляемому при выполнении контролируемых регламентных заданий
Куда сохранить скрипт Полный путь, включая имя файла, куда нужно записать сгенерированный скрипт
Допустимое отклонение Макс.количество минут отклонения временной метки в файле-индикаторе от текущего времени
Учетная запись почты Учетная запись почты с настроенным подключением к SMTP-серверу для отправки почты
Группа ответственных Список рассылки с адресами получателей уведомлений
Использовать аутентификацию ОС При установке соединения с SMTP-сервером из скрипта необходимо использовать параметры аутентификации операционной системы пользователя, запускающего скрипт. Логин и пароль для подключения к SMTP-серверу в скрипт записываться не будут.
Пароль к почте записать в скрипт Логин и пароль для подключения к SMTP-серверу записть в открытом виде в скрипт.
Текст письма-уведомления Текст письма, которое будет отправлено по списку рассылки получателей уведомления о неработающих регламентных заданиях.

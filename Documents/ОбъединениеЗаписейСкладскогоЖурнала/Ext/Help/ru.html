Документ предназначен для отражения в системе факта объединения записей складского журнала.

Документ «Объединение записей складского журнала» предназначен для объединения в ИС «Меркурий» нескольких записей складского журнала продукции в одну, для согласования особенностей партионного учёта, принятого в ИС «Меркурий» и в учётной системе предприятия. Также операция может использоваться для присоединения нескольких записей складского журнала к одной.

В результате успешного выполнения операции будет произведено добавление записи, сформированной в результате объединения двух или более записей журнала, содержащей общий объем, даты выработки и срок годности.

Для документа возможны 2 типа операции:

1.      Объединение – происходит объединение нескольких записей складского журнала продукции в одну, при этом добавляется новая запись складского журнала с общим объемом и периодом дат выработки и срока годности. Будет создана новая запись складского журнала со свойствами указанными в блоке «Данные результирующей записи».

2.      Присоединение – происходит присоединение записей к указанной записи складского журнала. В этом случае результирующая запись журнала будет иметь то же наименование продукции, что и указанная. Объём и количество упаковок будет просуммировано, вид упаковки, номера партий логически объединены в список.

Выполнение операции происходит от имени пользователя указанного в поле «Ответственный», поэтому у него должны быть указаны логин и пароль для работы с ИС «Меркурий» и у него должны быть права на обслуживание данного предприятия.

Тип операции «Объединение»

Для выполнения данного типа операции необходимо заполнить следующие данные:

1)      Данные шапки документа, по ним будут отобраны записи складского журнала, которые доступны для объединения:

-        Хоз. субъект – хозяйствующий субъект, на котором происходит выполнение операции;

-        Предприятие – предприятие, на котором происходит выполнение операции;

2)      Данные результирующей записи – это данные, на основании которых будет

сформирована новая запись складского журнала. Так же они являются критериями отбора, по которому происходит фильтрация записей доступных для объединения. На записи, которые возможно объединить (указываются в табличной части «Продукция») накладываются следующие ограничения:

-        Остаток по выбранному хоз. субъекту и предприятию должен быть больше 0;

-        Они должны принадлежать одному типу, виду, подвиду продукции указанной продукции в поле «Продукция»;

-        Страна происхождения должна совпадать с указанной в поле «Страна происхождения»;

-        Признаки «Скоропортящаяся продукция» и «Некачественный груз» так же должны совпадать;

-        Записи складского журнала не должны быть произведены на указанном предприятии, т.е. в результате выполнения «Производственной операции». Контроль данного условия пользователь производит самостоятельно.

Тип операции «Присоединение»

На выполнение данного типа накладываются те же ограничения, что и для типа «Объединение». Различие в том, что данные не указываются пользователем, а берутся из существующей записи складского журнала, которая указывается в поле «Запись складского журнала».

В результате успешного выполнения операции остатки с записей складского журнала указанных в табличной «Продукция» будут списаны в ноль и создана либо изменена запись складского журнала.
